Java多线程

Java多线程不管是Java Web开发、Java网络编程（服务端程序）还是说Android开发，几乎不可避免会用到多线程。

多线程初始

线程是指一个任务从头至尾的执行流。初学程序的时候，一般会以Hello World作为入门，而这一般离不开程序的入口Main函数，其实这就是启动一个线程。在Java中，每个任务都是Runnable接口的一个实例，也称为可运行对象（runnable object）。所以说线程本质上讲就是便于任务执行的对象。

创建任务和线程。

任务类（任务就是对象）：
TaskClass task = new TaskClass（….）；
1、按照上面的说法，每个任务都是Runnable接口的一个实例，则TaskClass就必须实现Runnable接口，Runnable接口只包含run方法，则只需要实现这个方法来告诉线程任务怎么执行。
Thread类（任务必须在线程中执行）：
Thread thread = new Thread（task）；
2、然后调用start方法告诉JVM来执行这个task任务。

Thread类

Thread -> java.lang.Thread -> 《interface》java.lang.Runnable

常用方法 描述
Thread() 创建一个空线程
Thread(task:Runnable) 为执行任务对象创建一个线程
start():void 启动线程是任务的run方法被JVM调用
isAlive():boolean 测试线程当前是否正在运行
setPriority(p:int):void 设置线程优先级（范围从1到10）
join():void 等待线程结束
sleep(millis:long):void 是线程睡眠指定的数
yield():void 是线程暂停并允许执行其他线程
interrupt():void 中断线程

刚开始的时候就说过，Thread实现了Runnable接口，因此，可以定义一个类继承Thread并override其run方法来达到创建任务并且启动线程的作用，但是这种方法将任务机制和线程运行机制混在一起，不符合设计理念。

继续上Demo：

这个Demo的场景是在线程先调用sleep后调用interrupt方法。则不难看出当程序在进行休眠的时候调用interrupt方法异常被捕获，sleep一下的语句不会被执行，捕获到异常后就return回去。当return删掉之后，catch外面的语句就会被执行，呢么30行就会输出。并且，27行输出时间=2000，一般不会差距太大，这说明在主线程睡眠2000ms的同时task线程也睡眠了2000s，当主线程休眠结束后，执行task线程的interrupt方法，task线程还在sleep，抛出异常。

场景二：线程先调用interrupt后调用sleep方法：同理，这调用interrupt后，run函数里面的sleep其实睡眠的时间为0，即使立刻捕获到异常并抛出。

interrupt方法只是让中断状态设置为true，当程序再次调用sleep、wait、join方法的时候，会捕获异常，同时中断状态再次设置为false，并且程序没有结束。如果想结束程序吗，可以将15行的注释删去，则18行就不会输出。

说到这里，大概对线程的中断有了比较基本的认识，不难得出一个结论：
调用interrupt后再调用sleep等3个方法或者说调用sleep等3个方法后调用interrupt方法的意思就在于，直接停止这个进程，return或者是线程继续运行，改变线程的状态，从暂停睡眠状态中唤醒。例如sleep（无限大：1年），这个线程将长时间睡眠下去，想要唤醒他，只能调用interrupt方法，让try-cagth捕获到异常，然后在catch块中转移到正常的逻辑上面。
